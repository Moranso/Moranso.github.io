git常用命令行

发表时间：2021-05-08 阅读： 评论：

git add 文件名 将文件放到暂存区

git add . 将所有修改文件放到暂存区

git commit -m “提交信息” 把暂存区文件提交到本地分支上

git checkout branchName 检出分支

git config –global ‘user.name’ ‘su’ 修改提交代码时的用户名

git commit –no-verify -m “ 绕过代码规范强制提交

版本回退

git status 查看仓库状态

git diff 查看修改的地方（如果git status显示有文件被修改 使用git diff）

git log —pretty=oneline 查看提交历史

git reset —hard HEAD~1 回退到第一个版本

git reset —hard 版本号 回退到指定的版本

git reflog 查看每一次的提交记录

小结：

  • HEAD指向的版本就是当前版本，因此，Git允许我们在版本的历史之间穿梭，使用命令git reset --hard commit_id。

  • 穿梭前，用git log可以查看提交历史，以便确定要回退到哪个版本。

  • 要重返未来，用git reflog查看命令历史，以便确定要回到未来的哪个版本。

git reset HEAD 文件名 把暂存区的文件，重新放回工作区

git checkout — 文件名 丢弃工作区文件中的修改

注意：

一种是文件自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

一种是文件已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。

git rm 文件名 删除仓库中的文件 然后需要重新提交一次

git checkout — 文件名 恢复删错的文件 （前提是该文件已经提交到仓库）

分支管理

git checkout -b dev 创建dev分支 并切换到dev分支

git branch dev 新建dev分支

git checkout master 切换到master分支

git branch 查看当前分支

git merge dev 合并dev到某分支（先切换到某分支 再把dev分支合并到那个分支）

git branch -d dev 删除dev分支

git stash 贮藏

git stash save “信息” 有贮藏信息

git stash list 查看贮藏列表

git stash apply stash@{0} 恢复指定贮藏文件

git stash drop 删除全部贮藏文件

git stash drop stash@{0} 删除指定贮藏文件

git stash pop 恢复贮藏 并把贮藏文件删除

git branch -D 分支名 强行删除一个没有被合并的分支

git remote 查看远程库的信息

git remote -v 查看远程库的详细信息

推送分支（把该分支上的所有本地提交推送到远程库）

git push origin 分支 推送分支到远程库

git checkout -b dev origin/dev 创建远程origin的dev分支到本地

拉取

git pull

注意

如果 git pull时 提示 no rracking information 则说明本地分支和远程分支的链接关系没有创建

git checkou -b branch-name origin/branch-name 在本地创建和远程分支对应的分支

git branch —set-upstream branch-name origin/branch-name 建立本地分支和远程分支的关联(没试过)

git branch –set-upstream-to=origin/远程分支的名字 建立本地分支和远程分支的关联(亲测可行)
